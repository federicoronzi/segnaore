SegnaOre — Design Spec

Revisiona la spec e dimmi se va bene nel terminale

Panoramica

App PWA per tracciare le ore lavorate giornalmente con resoconti settimanali, mensili e annuali consultabili e stampabili. Mobile-first, offline-first, dati salvati sul dispositivo (IndexedDB).

Stack: React + TypeScript + Vite + Tailwind CSS + Dexie.js

Flusso Giornaliero (4 step)

  1. Ora di Inizio — input ore:minuti grandi, bottone Avanti
  2. Ora di Fine — stessa struttura
  3. Pausa — 3 box: Sì (1h) / Mezza (30m) / No (0m)
  4. Cosa hai fatto — facoltativo, lista servizi con orario e descrizione

→ Riepilogo con ore lavorate in grande

→ Se la giornata è già compilata, mostra il riepilogo con bottone "Modifica"

Reperibilità

  • Toggle on/off per la settimana
  • Registra emergenze: solo data obbligatoria
  • Orario e descrizione facoltativi (sezioni apribili)
  • Contatore emergenze + ore totali (se orari inseriti)

Report

  • 4 periodi: Settimana / Mese / Anno / Custom
  • Navigazione con frecce ← →
  • Tabella giornaliera: inizio, fine, pausa, ore
  • Totale ore + straordinario automatico
  • Reperibilità inclusa nel report
  • Bottoni Stampa e Scarica PDF

Calendario

  • Vista mensile con ore per giorno
  • Colore diverso per straordinario
  • Icona emergenza se presente
  • Click su giorno → dettaglio + modifica

Impostazioni (wizard al primo avvio)

  1. Ore giornaliere standard (es. 8)
  2. Giorno con orario ridotto + quante ore (es. Venerdì = 6h)
  3. Giorni lavorativi (default Lun-Ven)

+ Esporta/Importa backup in JSON

Modello Dati

Tabella Contenuto
settings Ore standard, giorno ridotto, giorni lavorativi, flag setup completato
workEntries Una riga per giorno: inizio, fine, pausa, minuti lavorati, servizi
onCallWeeks Settimane di reperibilità attive
emergencies Emergenze: data (obblig.), orario e descrizione (facoltativi)
